Yllia Safety Classifier

A content classification tool for AI systems running on-premise, on the edge, or in air-gapped environments. The project focuses on Polish-language mental-health safety data and provides an open-source adapter that can be embedded into local AI workflows.

Project purpose

Open safety infrastructure for Polish AI systems.

Yllia Safety Classifier is not designed as a staffing automation tool. Its goal is to create a reusable, open-source Polish safety adapter that helps local AI systems recognize the type of user content before a downstream model, agent or RAG system decides how to respond.

01

Polish by design

Built around original Polish examples rather than treating Polish as a translated afterthought. English demo inputs are translated locally before classification; the adapter itself remains Polish-native.

02

On-prem and edge use

Designed for systems where sensitive content should stay inside the organization’s infrastructure, including local, air-gapped and edge deployments.

03

Reusable guardrail layer

Provides a classification layer that can sit before a chatbot, agent, intake form, RAG system or other AI application.

04

Open-source adapter

The intended output is a fine-tuned adapter that can be reused, audited and improved by the community.

Dataset

A Polish dataset with preserved class balance.

The training data started from 1,369 original questions. Synthetic examples were added only to the training split: one generated variant per original. The augmentation process preserved the original class frequency distribution, so the expanded training set remains aligned with the observed label proportions.

Taxonomy

Five labels for practical AI safety routing.

The classifier separates regular user input from off-topic content, medical-sensitive questions, crisis signals and adversarial or abusive messages. The labels are designed to support deterministic routing policies around an AI system.

Priority 1

ATTACK

Prompt injection, attempts to bypass rules, abusive language, spam, hostile documentation demands or attempts to break the intended procedure.

Priority 2

CRISIS

Signals of self-harm, suicidal ideation, acute psychological crisis or other content requiring special crisis handling.

Priority 3

MEDICAL_SENSITIVE

Medication, diagnosis, side effects, treatment decisions or other clinically sensitive content that should not be answered as ordinary information.

Priority 4

OK

Standard relevant messages that can be safely passed to a normal AI workflow, such as logistics, intake or general service questions.

Priority 5

OFFTOPIC

Messages unrelated to the intended mental-health or service context, including jokes, politics, weather or generic text generation requests.

Live demo

Try the local classification endpoint.

Enter a Polish message or let jurors type in English. English inputs are translated locally into Polish before classification, while the core adapter remains trained on Polish mental-health safety data. The demo expects a local backend endpoint at /api/demo.

English is supported through a local translation layer. The classifier adapter itself is Polish-native and trained on Polish data.

Pipeline: original input → local EN/PL handling → Polish classifier adapter → structured safety label.
