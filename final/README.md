# Yllia Safety Classifier — Gemma 4 E4B-it (LoRA adapter)

Polish-language input safety classifier for a psychiatric/psychological chatbot. A LoRA adapter fine-tuned on top of `unsloth/gemma-4-E4B-it`. Classifies a single user message into one of five categories and returns a short rationale as JSON.

**Author:** Damian Siwicki
**Repository:** https://github.com/shivihs/yllia-safety-classifier
**License:** Apache-2.0

## What it does

Tier-1 input triage in front of the main chatbot model. Each message is assigned one of five categories under a strict priority hierarchy (ATTACK > CRISIS > MEDICAL_SENSITIVE > OK > OFFTOPIC):

| Category | Description |
|---|---|
| `ATTACK` | Prompt injection, jailbreak attempts, abusive language, spam |
| `CRISIS` | Suicidal ideation, self-harm, acute psychosis, command hallucinations, violence |
| `MEDICAL_SENSITIVE` | Drug dosing, side effects, interactions, diagnosis/test interpretation requests |
| `OK` | Logistics and intake: appointments, pricing, scope of services, intent-to-seek-treatment, prescriptions |
| `OFFTOPIC` | Topics unrelated to the practice |

Output: `{"kategoria": "...", "uzasadnienie": "..."}`.

## Data

Polish-language dataset, ~3811 examples (stratified 80/10/10 split).

Pipeline: ~700 manually authored base questions → 5 synthetic variants each (short, emotional, typo-laden, verbose) generated by an LLM → category pre-labeling by an LLM with a hierarchy-enforcing prompt → human verification → targeted upsampling (+250 ATTACK, +200 CRISIS) → cleanup (API errors, duplicates, length outliers).

Train split: OK 1123 (36.8%) · OFFTOPIC 987 (32.4%) · MEDICAL_SENSITIVE 444 (14.6%) · CRISIS 338 (11.1%) · ATTACK 156 (5.1%).

## Known limitations

- **ATTACK recall (0.54).** More than half of undetected attacks split between OFFTOPIC and MEDICAL_SENSITIVE.
- **Over-cautious MEDICAL_SENSITIVE.** Emotional but logistical messages ("I'm angry", "nothing makes me happy") often land here instead of OK (18/96).
- **Isolated critical CRISIS misses.** Subtle suicidal cues embedded in logistics questions can be classified as OK.
- **False CRISIS.** Depressive but non-suicidal statements are sometimes escalated.
- **Generative rationale.** The `uzasadnienie` field is occasionally inconsistent with the assigned category — treat as a debug hint, not as an explanation of the decision.
- Polish only; single-message classification only; not a diagnostic or crisis-intervention system.

## Roadmap

A v2 dataset is being collected through a gamified Streamlit app with the AI4BUSINESS community as the data-collection partner. The focus is on expanding and diversifying the training data with real human-labeled examples to address the failure modes above.
